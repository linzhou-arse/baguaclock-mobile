# 📱 构建APK快速指南

## 🎯 推荐方案（按简单程度排序）

### ⭐⭐⭐ 方案1：GitHub Actions自动构建（最简单，推荐）

**优点**：无需本地环境，完全免费，自动构建

**步骤**：
1. 双击运行 `智能构建助手.bat`
2. 选择"1. GitHub Actions自动构建"
3. 按照提示上传代码到GitHub
4. 在GitHub上点击"Actions" → "Run workflow"
5. 等待构建完成，下载APK

**详细说明**：查看 `GitHub Actions构建指南.md`

---

### ⭐⭐ 方案2：WSL本地构建

**优点**：完全本地控制，可调试

**前置条件**：需要安装WSL

**步骤**：
1. 如果WSL未安装，运行 `手动启用WSL.bat`（需要管理员权限）
2. 重启电脑
3. 从Microsoft Store安装Ubuntu
4. 双击运行 `一键构建APK_完整版.bat`
5. 等待构建完成

**详细说明**：查看 `WSL构建说明.md` 和 `WSL安装指南.md`

---

### ⭐ 方案3：智能构建助手（自动检测）

**优点**：自动检测环境，选择最佳方案

**步骤**：
1. 双击运行 `智能构建助手.bat`
2. 根据检测结果选择方案
3. 按照提示操作

---

## 📋 文件说明

| 文件 | 用途 |
|------|------|
| `智能构建助手.bat` | 自动检测环境，选择最佳构建方案 |
| `一键构建APK_完整版.bat` | WSL环境下的完整构建脚本 |
| `一键构建APK.bat` | 原始构建脚本（兼容性更好） |
| `手动启用WSL.bat` | 手动启用WSL功能（需要管理员权限） |

| `.github/workflows/build_apk.yml` | GitHub Actions工作流配置 |

## ❓ 常见问题

### Q: 我应该选择哪个方案？

**A:** 
- **没有WSL，不想安装** → 选择GitHub Actions（方案1）
- **有WSL或愿意安装** → 选择WSL本地构建（方案2）
- **不确定** → 运行智能构建助手，自动检测

### Q: GitHub Actions需要付费吗？

**A:** 不需要！公开仓库完全免费。

### Q: WSL安装不了怎么办？

**A:** 
1. 查看 `WSL安装指南.md`
2. 或使用GitHub Actions方案（无需WSL）

### Q: 构建失败怎么办？

**A:** 
1. 查看错误信息
2. 参考对应方案的详细指南
3. 检查网络连接
4. 确保所有依赖已安装

## 🎉 完成后的下一步

构建成功后：

1. **找到APK文件**：通常在 `bin` 目录中
2. **安装到手机**：
   - USB安装：`adb install bin\app-debug.apk`
   - 直接传输：复制APK到手机，点击安装
3. **测试应用**：确保所有功能正常
